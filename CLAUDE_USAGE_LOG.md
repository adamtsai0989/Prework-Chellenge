## Project Planning(PRD was created)

**1.** Initial project request
> "Create a simple web application of "Two truth one lie" game for team building in react"

**2.** Game design clarifications (response to questions about users, game flow, real-time vs async, and constraints)
> "1. 5-10 players team building game for colleagues
> 2. Player on the stand will submit their responses including two truth and one lie. Other players guess which statement is a lie. Round ends when player on the stand reveals the answer. Points awarded to players with right guess.
> 3. Game should be real-time, like a live meeting game.
> 4. Backend should be designed in express.js. The app should also be able to function in an offline setting (passing the device around)"

**3.** Resolving PRD open questions (round 1)
> "1. Offline mode should handle state in local storage only
> 2. The join code should expire after each game session ends
> 3. Automatically promote another player to host if the host disconnects mid-game in real-time mode
> 4. The app should remain work appropriate, this applies to any entered name or submission
> 5. The reveal animation should be handled on client-side (css)
> 6. There should be a toggle for entering debugging or testing mode that does not have a player count requirement for real-time mode"

**4.** Resolving PRD open questions (round 2)
> "1. Content filter should only include common profanity
> 2. Auto-promoted host should be next player in join order
> 3. Debug mode should be accessible via URL flag only
> 4. Activity timeout should be 45 seconds"
